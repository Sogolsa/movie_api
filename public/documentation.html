Documentation Page

The movie API is a REST API for an application called myFlix that interacts with a database that stores data about different movies. It allows users to search for movies, get details about specific movies, and more. This document aims to guide developers on how to use the API endpoints efficiently.

API ENDPOINT DOCUMENTATION

List Number Request Method/CRUD URL Endpoint Request Response
1 GET/READ / Return a welcome message. A default text
2 GET/READ /movies Return a list of all movies to the users. A JSON object holding data about all the movies.
3 GET/READ /movies/:title Return data about a single movie by title to the user. A JSON object holding the data about a single movie by title, or a text that shows the movie doesn't exist.
4 GET/READ /movies/genres/:genreName Return data about a genre by name title. A JSON object holding the data about the genre, or a text that shows the genre doesn't exist.
5 GET/READ /movies/director/:directorName Return data about a director by name. A JSON object holding the movie by director name, or a text that shows the director doesn't exist.
6 POST/CREATE /users Allow new users to register. A JSON object holding data about new user, or a text that shows user needs a name.
7 PUT/UPDATE /users/:Name Allow new users to update their user's info, specially username. A JSON object with updated user name, or a text that shows "no such user!".
8 POST/CREATE /users/:Name/movies/:MovieID Allow users to add a movie to their list of favorites. A text that a movie has been added.
9 PUT/UPDATE /users/:Name/Movies/:MovieID Allow users to remove a movie from their list of favorites. A text that a movie has been removed.
10 PUT/UPDATE /users/:Name Allow existing users to deregister. A text that a user email has been removed.
